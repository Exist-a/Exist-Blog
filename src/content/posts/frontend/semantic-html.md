从学前端到现在，你用的最多的想必就是 div 盒子了吧。从上到下，从左到右，只要拿盒子套一套，给些颜色、文字和大小，就可以生成出一个大体框架，接着你只需要在此基础上进行修改即可。这固然方便，但你或许没意识到：比起一味使用 div，在对应位置用对专属标签，才能让网页质量更上一层楼。

在众多的 HTML 标签里，有一些我们可以通过它的名字知道它是干什么用的，例如：`<nav>` 对应的就是导航（navigation），`<article>` 对应的就是文章，`<title>` 对应标题。这些标签我们就称之为"语义化标签"。

<br />

接下来重要的来了，**我们为什么要使用语义化标签呢？**

## 1. 利于呈现更好的代码结构

由于语义化标签的特点就是**带有语义**，所以语义化代码也能体现更好的逻辑性，具体区别一看便知：

**语义化写法**

```html
<body>
  <header>header</header>
  <nav>nav</nav>
  <main>
    <article>article</article>
    <section>section</section>
  </main>
  <footer>footer</footer>
</body>
```

**纯 div 堆叠写法**

```html
<body>
  <div>header</div>
  <div>nav</div>
  <div>
    <div>article</div>
    <div>section</div>
  </div>
  <div>footer</div>
</body>
```

在对代码进行优化、修改时，语义化也会为人们降低极大的阅读成本。

## 2. 有利于 SEO 优化

MDN 对于 SEO 优化是这样描述的：**SEO**（搜索引擎优化）是一种让网站在搜索引擎结果中更加清晰，也帮助我们将搜索结果更靠前的过程。

概括来说就是搜索引擎会"看"你的网页结构，根据你的结构为你的网页打分，分数越高，你的网页排名就会越靠前，人们搜索到你的网页的概率就越大。而多使用网页语义化就有利于提高你的排名。

由于 SEO 是靠爬虫爬取网页内容来分析判断的，而爬虫对代码的**理解能力**依赖于清晰的结构标识——如果网页全是无意义的 `<div>` 嵌套，爬虫需要反复解析自定义的 `class` 名才能勉强推断内容逻辑，既容易遗漏核心信息，也可能因命名混乱产生误判；但语义化标签能直接给内容"贴好身份标签"，让爬虫快速识别核心内容，大幅降低解析成本，自然会让网页在 SEO 打分中更占优势。

## 3. 方便其他设备解析

而语义化标签的核心价值，不仅在于优化开发者体验和 SEO，更在于为**非浏览器类访问设备**提供清晰的内容解读规则。这些设备（如屏幕阅读器、智能音箱、车载系统等）无法像人一样浏览网页，只能依赖 HTML 标签的语义信息来理解内容结构与含义，而语义化标签解决了设备读不懂内容的问题，让不同需求的用户都能顺畅获取信息，实现真正的"无障碍访问"与"多端适配"。

总之，div 虽能帮我们快速搭起网页框架，但真正优质的前端开发，从来不止于"能用"，更在于"好用"与"通用"。语义化标签看似只是换了种标签写法，实则是为网页注入了清晰的逻辑骨架。它让代码更易读、让搜索更易找到、让更多设备与用户都能顺畅获取信息。学会在对应位置用对语义化标签，不仅是技术能力的提升，更是对网页质量与用户体验的深层考量，也是前端开发中值得坚持的良好实践。
